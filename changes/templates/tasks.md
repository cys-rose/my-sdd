## 1. 使用规则

1. 本文件是 `/apply` 阶段的治理记录。
2. 没有对应 `spec.md` 和 `plan.md` 时，不得执行。
3. 不得跨 Task 偷跑；若发现范围变化，先回写 `spec.md`。
4. 每个 Task 完成后必须在此记录验证证据与文档同步情况。

## 2. 执行前检查

- [ ] 已阅读 `spec.md`
- [ ] 已确认 `plan.md` 已生成
- [ ] 已确认本次只做当前变更范围
- [ ] 已确认风险点与业务规则
- [ ] 已确认用户允许进入 `/apply`
- [ ] 已确认需要同步更新的文档
